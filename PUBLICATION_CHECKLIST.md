# Publication Checklist

Use this before creating the public GitHub repository.

## Repository Safety

- [ ] No production source code included.
- [ ] No private repository history included.
- [ ] No `.env`, secrets, keys, tokens, certificates or credentials.
- [ ] No Supabase service-role logic or sensitive policy detail.
- [ ] No admin routes or private operational notes.
- [ ] No private user, artist, fan, mailing-list or analytics data.
- [ ] No protected DCE game or band IP.
- [ ] No unclear third-party copyrighted assets.

## Wording Safety

- [ ] Claims are accurate and supportable.
- [ ] AI-assisted development is described honestly.
- [ ] The case study does not overclaim senior expertise in every stack item.
- [ ] Commercially sensitive mechanics are kept high-level.
- [ ] The repo explains why source code is private without sounding evasive.

## Recruiter Usefulness

- [ ] README explains what Metopus is in plain English.
- [ ] README explains Alec's role clearly.
- [ ] Tech stack is visible.
- [ ] Architecture is understandable in under one minute.
- [ ] Docs show product thinking, not just feature lists.
- [ ] Screenshots or diagrams are added only after review.

## Links To Add Later

- [ ] Live Metopus public site link.
- [ ] Personal portfolio link after update.
- [ ] LinkedIn link.
- [ ] Selected screenshots.
- [ ] Optional short screen recording or GIF.
